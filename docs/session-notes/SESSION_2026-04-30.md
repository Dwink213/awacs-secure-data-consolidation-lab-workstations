## Honest Gaps (named per user's "ok to fail" authorization)

1. **38 of 52 test specs have no executable counterpart.** The runnable subset covers the most load-bearing checks (storage hardening, KV hardening, immutability, diag settings, consumer RBAC). The rest are spec-only and listed in `tests/scripts/README.md` as known TODOs.
2. **No live deployment was run.** Bicep is structurally complete and references real ARM API versions, but it has not been `az deployment what-if`'d or `az deployment group create`'d against a real subscription. Field validation will surface the usual class of issues (parameter typos, scope-construction edges, null-handling on optional outputs). The deploy script paths are best-effort.
3. **SAS rotator not implemented.** RUNBOOK.md documents both the manual rotation command and the recommended Function-based rotator. Day-1 deploys work because Deploy.ps1 seeds an initial 24h SAS; day-2+ requires the operator to run rotation (or wire the rotator).
4. **Mode B cert distribution (external PKI) documented but not coded.** Deploy.ps1 implements Mode A only.
5. **PEM cert import in bootstrap.ps1 has a known limitation.** PowerShell 5.1 cannot import a PEM-with-private-key directly; bootstrap detects this and instructs the operator to convert to PFX with openssl. PFX import works natively.
6. **No subscription-level diagnostic setting deployed by default.** Component 05 mentions it as a Stage-5 wiring concern but the Bicep at `deploy/main.bicep` does not include the subscription-scope diag (it would require `targetScope = 'subscription'`). Operator can add via standalone command per RUNBOOK.md.
7. **Cross-agent disagreements directory is empty.** The four agents agreed on every signed artifact in this run. Future iterations that surface disagreements will populate `docs/cross-agent-reviews/`.
