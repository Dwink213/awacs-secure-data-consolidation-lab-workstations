## Primary Finding: IaC-Reality Inversion in ADR-008

**The inversion (verbatim from ADR-008 before fix):**
- Decision section: "Add an Azure Function (component 08)…"
- Alternatives Considered table: "Azure Automation runbook — Viable but… adds less cognitive overhead. [Rejected.]"

**Live reality:** The system runs an Azure Automation Account. No Function App exists.

**Root cause:** The ADR was written during the planning phase (Azure Functions was Plan A). The pivot to Automation Account happened mid-build when the quota wall was hit. The ADR was never updated after the pivot. The document was well-structured, version-controlled, and confidently wrong.

**Named anti-pattern:** IaC-Reality Inversion — a documentation failure mode in which the ADR records the planned approach as chosen and the actual shipped approach as rejected. More dangerous than a missing ADR because it reads as authoritative.
