How It Works

Five Steps. Zero Trust.

Every credential is time-limited. Every action is logged. Every boundary is enforced.

STEP 01
🖥
Lab Workstation

Analyst saves data files. A scheduled task fires every 30 minutes automatically — no manual action required.

On-Prem · Low Trust
No admin rights
Service account only
Cert stored in OS cert store
TLS 1.2+
STEP 02
🔐
Identity Check

SP certificate proves machine identity to Azure Entra ID. Private key never leaves the workstation — JWT signed in-place.

Cert Only · No Password
Private key never leaves
Access token: 1h lifetime
Revocable independently
Token (1h)
STEP 03
🔑
Key Vault

Token unlocks today's write-only SAS token from Key Vault. SAS expires every 24 hours and rotates automatically.

Azure · High Trust
Write-only SAS: sp=acw
No read · No list · No delete
Rotated daily, auto-expired
Write-Only SAS
STEP 04
🔒
Secure Storage

File written once. Azure WORM policy locks it for 90 days. Even the storage owner cannot delete it before the lock expires.

Immutable · Audited
Azure-enforced retention
Versioning + soft delete on
Every write logged to Log Analytics
Read-Only RBAC
STEP 05
👤
Analyst Access

MFA-authenticated individual identity. Read-only RBAC enforced at the platform layer — immutability blocks delete even if RBAC misconfigured.

Med Trust · Logged
Cannot write or delete
Read log → Log Analytics
Immutability blocks any override
0 Passwords
Write-Only Access
90-Day WORM
100% Logged
Cert-Only Auth
Problem & Solution

What This Solves

Shared lab environments are an unsolved data governance problem. This changes that.

Before — The Problem

  • ❌Sensitive files leave via USB, email, or shared drives
  • ❌No audit trail of who accessed or copied what
  • ❌Shared login credentials — no individual accountability
  • ❌Data can be altered or deleted after collection
  • ❌Manual process — dependent on analyst compliance

After — This System

  • ✅Automated push every 30 minutes — no human in the loop
  • ✅Every auth event, file write, and access logged in Azure
  • ✅Certificate auth — individual machine identity, no shared secrets
  • ✅WORM-locked for 90 days — tamper-proof by platform policy
  • ✅Pull-down deployable — one command, no manual portal steps
