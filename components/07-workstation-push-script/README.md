# Component 07 — Workstation Push Script

**Atomic Lego.** Single responsibility: the runtime push from the lab PC.

This component is workstation-side. The script itself is `workstation/push-files.ps1`. The bootstrap that installs prerequisites is `workstation/bootstrap.ps1`. The configuration the script reads is `workstation/config.json` (templated at deploy time).

This component README documents the *contract* the script must satisfy.

## Runtime contract

On every scheduled invocation, the script:

1. Loads config from `C:\ProgramData\AwacsBackup\config.json`
2. Loads the SP cert by thumbprint from `Cert:\CurrentUser\My`
3. Authenticates to Entra ID as the SP using `Connect-AzAccount -ServicePrincipal -CertificateThumbprint <thumb> -ApplicationId <appid> -Tenant <tenant>`
4. Calls `Get-AzKeyVaultSecret -VaultName <kv> -Name current-write-sas` to retrieve today's SAS
5. Enumerates files under `C:\labdata\` (configurable) that are not in the local `pushed.json` ledger
6. For each unpushed file:
   - Computes SHA-256
   - Constructs blob name: `<hostname>/<YYYY-MM-DD>/<filename>`
   - PUTs to storage using SAS
   - On 201, appends to `pushed.json` ledger
7. Self-checks cert expiry; if <14 days, writes synthetic `_health/<hostname>-cert-expiring.json`
8. Logs every step to `C:\ProgramData\AwacsBackup\logs\push-YYYY-MM-DD.log` via `Start-Transcript`
9. Exits 0 on success, non-zero on any error

## Logging contract (CLAUDE.md Rule 8)

- Levels: INFO, WARN, ERROR, CRITICAL
- Format: `[YYYY-MM-DDTHH:MM:SS] [LEVEL] message`
- SAS string never appears in logs (regex-redacted)
- Cert thumbprint logged in INFO at startup
- File paths logged in INFO; file contents never logged
- Errors logged with full exception, including stack
- Transcript file rotated daily; 14-day retention configurable

## Why no Bicep

This component is a script, not an Azure resource. Its "deployment" is the bootstrap script running on each lab PC. There is nothing in ARM to declare.

## Code-signing path

For production, `push-files.ps1` should be Authenticode-signed. The signing procedure:

1. Operator obtains a code-signing cert from internal PKI or commercial CA
2. Run: `Set-AuthenticodeSignature -FilePath workstation/push-files.ps1 -Certificate <cert>`
3. Lab PC's `Trusted Publishers` store includes the signing cert's issuer

For dev/test, `Set-ExecutionPolicy RemoteSigned` permits local-origin unsigned scripts. For prod, `AllSigned` is the target.
